# Zom 编辑引擎性能基线

版本：`v1.0`  
适用范围：`/Users/liuyuan/project/zom`  
目标：为编辑引擎核心路径提供可重复、可对比、可门禁的性能基准。

---

## 1. 执行命令

1. 普通模式（仅输出结果，不阻断）  
`./scripts/bench-editor-core.sh`
2. 门禁模式（超阈值退出非 0）  
`./scripts/bench-editor-core.sh --enforce`

---

## 2. 基线场景

1. `insert_tail_1mb`：1MB 文本尾部连续插入。
2. `insert_tail_5mb`：5MB 文本尾部连续插入。
3. `delete_backward_tail_1mb`：1MB 文本尾部连续退格删除。
4. `move_right_1mb`：1MB 文本连续右移光标。
5. `offset_position_roundtrip_5mb`：5MB 文本 `offset <-> position` 映射往返。
6. `wrap_visual_line_1mb(ms)`：1MB 长行软换行耗时。

---

## 3. 当前阈值（门禁）

1. `insert_tail_1mb`：`p95 <= 1500us`
2. `insert_tail_5mb`：`p95 <= 3000us`
3. `delete_backward_tail_1mb`：`p95 <= 1500us`
4. `move_right_1mb`：`p95 <= 600us`
5. `offset_position_roundtrip_5mb`：`p95 <= 200us`
6. `wrap_visual_line_1mb`：`<= 350ms`

---

---

## 5. 使用约定

1. 变更 `zom-text`、`zom-editor`、`zom-runtime` 核心路径时，至少跑一次普通模式。
2. 发版前或合入高风险编辑改动时，跑门禁模式。
3. 若业务需要提高阈值，必须在 PR 里附性能对比与原因说明，并同步更新本文件。
