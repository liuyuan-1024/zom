# Zom 编辑引擎核心能力开发清单

版本：`v1.0`  
适用范围：`/Users/liuyuan/project/zom`  
目标：围绕“稳定可用、可验证、可演进”的编辑引擎核心能力，建立持续交付清单。

---

## 1. 范围与边界

1. 本清单只覆盖编辑引擎核心能力，不包含 IDE 外围效率功能（如补全面板、主题系统、项目级工具链）。
2. 能力落点遵循既有架构约束：语义在 `zom-protocol`，算法在 `zom-text`/`zom-editor`，编排在 `zom-runtime`，渲染在 `zom-gpui`。
3. 所有能力项必须满足“可演示 + 可自动化验证 + 可回滚”。

---

## 2. 核心能力清单（按优先级）

### M0：语义与命令主链（必须先完成）

1. 能力项
   - 定义最小编辑语义集：插入、删除、移动、选区、撤销、重做。
   - 建立 `zom-protocol -> zom-input -> zom-runtime -> zom-editor` 统一命令链路。
   - 保证键盘输入、命令调用、状态回写的链路可追踪。
2. 验收标准
   - 最小语义集有稳定 ID 与元数据，且能端到端触发。
   - 核心编辑命令集成测试全通过。
3. 主要落点
   - `crates/zom-protocol`
   - `crates/zom-input`
   - `crates/zom-runtime`
   - `crates/zom-editor`

### M1：文本模型与位置映射正确性（必须）

1. 能力项
   - 提供稳定的插入、删除、查询、位置映射 API。
   - 明确字符边界语义：grapheme、emoji、组合字符、换行边界。
   - 建立文本边界测试矩阵（普通文本、CJK、混合符号、多行场景）。
2. 验收标准
   - 文本模型边界用例全部通过。
   - 连续编辑 10 分钟无错位、无丢字、无重复插入。
3. 主要落点
   - `crates/zom-text`
   - `crates/zom-editor`

### M2：选区/光标/事务与撤销重做（必须）

1. 能力项
   - 实现“单次用户意图 -> 单次可撤销事务”模型。
   - 实现输入合并与事务断点策略。
   - 统一 IME（中英文）与普通键盘输入下的撤销行为。
2. 验收标准
   - 撤销重做连续 200 次行为正确。
   - IME 场景下不出现光标跳变、事务错并、撤销错乱。
3. 主要落点
   - `crates/zom-editor`
   - `crates/zom-runtime`

### M3：渲染与大文档性能（必须）

1. 能力项
   - 可视区增量渲染与滚动平滑。
   - 编辑后仅刷新受影响区域。
   - 支持最小高亮能力（纯文本 + 一种代码语言）。
2. 验收标准
   - 1MB 文本文件场景：输入延迟、滚动延迟、内存占用均不劣于基线。
   - 编辑过程中无整屏闪烁。
3. 主要落点
   - `crates/zom-gpui`
   - `crates/zom-runtime`

### M4：文件生命周期闭环（必须）

1. 能力项
   - 打通打开、保存、自动保存、崩溃恢复。
   - 支持 UTF-8 与 LF/CRLF 识别和保留策略。
   - 支持单文件查找替换（大小写、全词、正则）。
2. 验收标准
   - 意外退出后可恢复未保存草稿。
   - 查找替换行为与主流编辑器抽样一致。
3. 主要落点
   - `crates/zom-runtime`
   - `crates/zom-editor`
   - `apps/zom-desktop`

### M5：可观测与回滚门禁（必须）

1. 能力项
   - 关键路径打点：输入延迟、渲染耗时、撤销失败率、恢复成功率。
   - 为新增能力提供降级开关或安全回退路径。
   - 建立每周基线对比与回归报告。
2. 验收标准
   - 每次合入可定位性能与行为回归。
   - 任一新增能力可在运行时关闭或回退。
3. 主要落点
   - `crates/zom-runtime`
   - `apps/zom-desktop`

---

## 3. 开发顺序与门禁

1. 顺序要求：`M0 -> M1 -> M2 -> M3 -> M4 -> M5`。
2. 门禁要求：前一层未通过验收，不进入下一层。
3. 并行要求：允许并行开发，但不得绕过门禁直接上线。

---

## 4. 统一验收口径（每项都要过）

1. 功能验收：能力项可端到端演示。
2. 测试验收：新增能力必须有自动化测试。
3. 性能验收：关键指标不低于最新基线。
4. 回滚验收：能力可关闭、可降级、可恢复。

---

## 5. 风险与应对

1. 事务一致性风险（undo/redo 错乱）  
   - 应对：事务行为矩阵作为回归门禁。
2. 大文件性能风险（输入和滚动卡顿）  
   - 应对：建立固定数据集与基线压测任务。
3. 文本边界风险（Unicode/IME 异常）  
   - 应对：把边界样本纳入 CI 必跑用例。
4. 范围膨胀风险（核心能力被外围需求打断）  
   - 应对：外围需求统一进入 backlog，不阻塞核心主线。

---

## 6. 完成定义（Definition of Done）

满足以下条件，可视为“编辑引擎核心能力完成”：

1. M0~M5 全部通过统一验收口径。
2. 可支撑日常文本编辑主流程并稳定运行。
3. 回归问题可定位、可回滚、可持续迭代。
