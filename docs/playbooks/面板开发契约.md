# 面板开发契约（文档层）

版本：`v0.5`  
状态：`Active`（当前面板开发 SoT）  
适用范围：`zom-protocol`（含 `zom-protocol::input`）、`zom-runtime`、`zom-gpui`（可扩展到 `zom-workspace`）

---

## 1. 目标

本契约用于统一“一个面板从协议到交互”的开发方式，避免每新增一个面板都重复设计一套输入、焦点、显隐和分发逻辑。

本契约聚焦三件事：

1. 明确跨 crate 协作边界，降低跳转和重复实现。
2. 固化面板生命周期和输入焦点语义，避免行为漂移。
3. 提供可复用 checklist，让 Git/Outline/Search/Terminal 等面板按同一模板推进。

---

## 2. 术语与对象

1. `Panel`：受 `FocusTarget::VISIBILITY_MANAGED_PANELS` 管理的工作台侧边面板。
2. `Show + Focus`：显示面板并让该面板成为当前焦点目标。
3. `Hide Focused Panel`：隐藏当前焦点面板并把焦点回退到 `Editor/Pane`。
4. `Panel Scoped Command`：仅当焦点在该面板时才生效的命令（如文件树上下移动）。

---

## 3. 协作边界（跨 crate 契约）

### 3.1 `zom-protocol`（协议层）

1. 定义面板身份：`FocusTarget::XxxPanel`。
2. 定义命令语义：`CommandKind` + `CommandInvocation`，以及 `WorkspaceAction::{FocusPanel, CloseFocused, Xxx(XxxAction)}`。
3. 在 `command::kind` 的 `CommandKindSpec` 中维护命令稳定 Key、命令描述、默认快捷键元数据。
4. 保持命令协议稳定，并在 `zom-protocol::input` 承载平台无关的默认输入解析能力。

### 3.2 `zom-protocol::input`（输入映射层）

1. 定义默认键位映射。
2. 默认键位应由 `zom-protocol::command::default_shortcut_bindings` 生成，避免手写散落。
3. 负责 `Keystroke + InputContext -> CommandInvocation` 的绑定与作用域匹配。
4. 输出 `InputResolution::Command(CommandInvocation)`。
5. 不落地业务状态，不直接操作 UI 实体。

### 3.3 `zom-runtime`（编排层）

1. 维护面板可见性集合：`visible_panels`。
2. 维护统一焦点状态：`focused_target` + `pending_focus_target`。
3. 统一命令分发：`handle_keystroke -> handle_command -> handle_workspace_command`。
4. 持有面板领域状态：如 `FileTreeState`。
5. 提供命令展示投影接口（如 `zom-runtime::projection::shortcut_hint`），统一把命令语义投影给 UI。

### 3.4 `zom-gpui`（渲染与事件适配层）

1. 把平台事件转换成 `zom-protocol::Keystroke`。
2. 根视图统一收敛键盘事件，调用 `zom-runtime`。
3. 根据 `visible_panels` 决定是否挂载面板组件。
4. 根据 `pending_focus_target` 回放焦点到具体 View。
5. tooltip/菜单快捷键提示只消费 `zom-runtime::projection`，禁止直接读取 `zom-protocol` catalog 展示投影。

### 3.5 `zom-workspace`（可选承载层）

当面板数量增长导致 `DesktopAppState` 过重时，优先将“通用面板生命周期能力”下沉到 `zom-workspace`，业务语义留在 `zom-runtime`。

### 3.6 通用规则引用（SoT）

1. 命令命名、术语与 `CommandKindId` 模板，以 [开发规范手册](../standards/开发规范手册.md) 为准。
2. 注释与 rustdoc 要求，以 [开发规范手册](../standards/开发规范手册.md) 为准。
3. 边界执行纪律、allowed/forbidden 依赖，以 [Crate边界契约](../architecture/Crate边界契约.md) 为准。
4. 本文档不重复定义上述通用规则，只保留面板专属约束。

### 3.7 面板专属增补（本文件 SoT）

1. 每个面板必须提供 `FocusPanel(XxxPanel)` 语义入口，并复用全局 `CloseFocused`。
2. 所有 `Panel Scoped Command` 必须受 `focused_target == XxxPanel` 守卫。
3. 面板显隐、焦点回放、快捷键提示三条链路必须同时打通：`visible_panels`、`pending_focus_target`、`zom-runtime::projection`。
4. 面板激活对象后若回焦编辑区，必须显式走 runtime 的焦点回退路径，禁止组件层绕过 runtime 私改焦点状态。

## 4. 面板生命周期契约

### 4.1 标准状态流

1. `Hidden`
2. `Visible + Focused`
3. `Visible + Unfocused`

### 4.2 标准动作语义

1. `FocusPanel(XxxPanel)`：确保可见，并切换焦点到该面板。
2. `CloseFocused`：关闭当前聚焦组件。若当前聚焦对象是可见面板，则隐藏并回焦 `Editor`；若当前聚焦对象是编辑区，则关闭当前激活标签页。
3. `PanelScopedCommand`：仅在当前 `focused_target == XxxPanel` 时解析并执行。
4. 文件型面板激活对象后（如文件树打开文件），允许业务层把焦点主动回退到 `Editor`。

### 4.3 不变量

1. 不允许“隐藏面板但焦点仍指向该面板”。
2. 不允许组件层私自维护独立命令分发逻辑，必须回到 `zom-runtime` 统一入口。
3. 面板显隐策略与焦点策略必须可测试，不依赖“视觉上看起来对”。

---

## 5. 输入与快捷键契约

### 5.1 统一策略

1. 每个面板有一个全局 `Show + Focus` 快捷键。
2. 所有面板共享一个“关闭当前聚焦组件”快捷键（建议 `Cmd+W`，全局绑定到 `CloseFocused`）。
3. 面板内部导航/操作快捷键使用 scoped 绑定，不污染全局。
4. 低频命令优先走命令面板，不新增全局键位复杂度。

### 5.2 文案一致性

1. `zom-gpui` tooltip 必须通过 `zom-runtime::projection::shortcut_hint(CommandInvocation)` 查询快捷键文案。
2. 若键位策略调整，必须同 PR 更新：
键位映射代码、tooltip 文案、对应测试断言。
3. 禁止在组件里硬编码快捷键字符串（如 `Cmd+...`），统一走 `zom-runtime::projection::shortcut_hint(CommandInvocation)`。

## 6. 开发清单（新面板模板）

### 6.1 协议层（`zom-protocol`）

1. 在 `FocusTarget` 新增 `XxxPanel`。
2. 在 `WorkspaceAction` 增加 `Xxx(XxxAction)`（按需求）。
3. 在 `command::kind` 的单一 `CommandKindSpec` 声明源中新增 `CommandKind`、`CommandMeta`、默认快捷键定义。
4. 为 `XxxAction` 提供 `From` 到顶层 `CommandInvocation` 的转换。

### 6.2 输入层（`zom-protocol::input`）

1. 确认 `default_shortcut_bindings` 已覆盖该面板对应命令。
2. 校验 `Show + Focus` 的全局快捷键生效。
3. 校验 `CloseFocused` 的全局快捷键生效。
4. 校验面板内部 scoped 快捷键生效。
5. 补充默认 keymap 解析测试。

### 6.3 编排层（`zom-runtime`）

1. 新增 `XxxState`（按领域数据组织）。
2. 在 `DesktopAppState` 中新增 `handle_xxx_command`。
3. 接入 `handle_workspace_command` 分发。
4. 落地焦点回退/切换规则。
5. 补充“显示/隐藏/聚焦/空态”测试。

### 6.4 渲染层（`zom-gpui`）

1. 新建 `XxxPanel` 组件（`new`、`set_state`、`Focusable`、`Render`）。
2. 根视图根据可见性挂载/卸载面板。
3. 在 `sync_child_views` 同步对应状态。
4. 在 `apply_focus_target` 处理 `XxxPanel` 焦点回放。

### 6.5 可选演进（`zom-workspace`）

1. 抽取通用 `PanelManager`（可见性、焦点、快捷键策略）。
2. `zom-runtime` 仅持有业务态与业务动作，减少状态机膨胀。

## 7. 测试契约

### 7.1 必测项

1. `core`：命令提升与协议语义测试。
2. `core::input`：全局键位与 scoped 键位解析测试。
3. `app`：显隐 + 聚焦 + 回焦 + 空态测试。
4. `gpui`：最少保证焦点回放路径不丢（可先从集成层验证）。

### 7.2 回归优先级

1. 快捷键改动必须有回归测试。
2. 焦点策略改动必须有回归测试。
3. 显隐策略改动必须覆盖“Pane 为空”的场景。

---

## 8. PR 验收清单

1. 是否遵守了 `core::input -> app -> gpui` 单向职责？
2. 是否遵守了“core 存规范，app 做投影，gpui 只消费投影”？
3. 是否只有一个主 crate 承载本次功能核心？
4. 是否补齐了键位文案与映射的一致性更新？
5. 是否覆盖了“显示/隐藏/聚焦/空态”测试？
6. 是否避免在 `gpui` 里写命令映射业务逻辑？
7. 是否满足 rustdoc 注释要求（模块 `//!`、公开类型/函数/核心字段 `///`，并正确标注 `TODO/FIXME`）？
8. 是否存在边界模糊或职责漂移（若有，是否已在本 PR 先修复）？
