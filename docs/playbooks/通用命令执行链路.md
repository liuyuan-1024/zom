# Zom 通用命令执行链路

版本：`v0.2`  
适用范围：`/Users/liuyuan/project/zom`  
目标：用“函数名视角”描述从输入到状态更新再到 UI 回显的统一命令链路，便于后续新增命令时快速定位改动点。

---

## 1. 核心模型

1. 命令语义层（稳定）
`CommandKind` + `CommandKindSpec`
入口函数：`command_kind_specs`、`command_kind`、`command_meta`  
所在文件：`crates/zom-protocol/src/command/kind/mod.rs`

2. 命令调用层（运行时）
`CommandInvocation`（可携带动态 payload）
入口类型：`CommandInvocation::Editor`、`CommandInvocation::Workspace`  
所在文件：`crates/zom-protocol/src/command/invocation/mod.rs`

3. 输入解析层
`Keystroke -> InputResolution(Command | InsertText | Noop)`
入口函数：`resolve_default`  
所在文件：`crates/zom-input/src/lib.rs`

---

## 2. 标准执行链路（键盘路径）

### 阶段 A：GPUI 事件进入根视图

1. `ZomRootView::render` 注册按键监听。
2. `ZomRootView::handle_shortcut_keydown` 作为统一键盘入口。
3. `to_core_keystroke` 负责把 GPUI 事件转成协议按键。

对应文件：
`crates/zom-gpui/src/root_view/render.rs`  
`crates/zom-gpui/src/root_view/mod.rs`  
`crates/zom-gpui/src/input.rs`

### 阶段 B：输入策略层做按键解析

1. `DesktopAppState::handle_keystroke` 调用输入策略层解析。
2. `resolve_default` 先走默认 keymap。
3. `Keymap::resolve` 先查焦点作用域，再回退全局作用域。
4. `resolve_editor_text_fallback` 处理“可直接插入文本”的兜底场景。

对应文件：
`crates/zom-runtime/src/state/desktop_app.rs`  
`crates/zom-input/src/lib.rs`  
`crates/zom-input/src/keymap.rs`

### 阶段 C：命令分发

1. `DesktopAppState::handle_command` 是顶层命令总入口。
2. `DesktopAppState::handle_workspace_command` 处理工作台域。
3. `DesktopAppState::handle_editor_command` 处理编辑器域。

对应文件：
`crates/zom-runtime/src/state/desktop_app.rs`

### 阶段 D：领域执行

1. 编辑器路径：`apply_editor_invocation`。
2. 动作分派：`apply_action`。
3. 选区/光标移动：`apply_selection_move`。
4. 事务提交：`apply_with_selection`。

对应文件：
`crates/zom-editor/src/features/editing/invocation.rs`

### 阶段 E：状态回写与 UI 同步

1. Runtime 侧更新工具栏光标：`sync_tool_bar_with_active_tab`（使用 `selection().active()`）。
2. Root 侧下发最新状态：`sync_child_views`。
3. Pane 侧更新内部状态：`PaneView::set_state`。

对应文件：
`crates/zom-runtime/src/state/desktop_app.rs`  
`crates/zom-gpui/src/root_view/mod.rs`  
`crates/zom-gpui/src/components/pane/view.rs`

### 阶段 F：选区与光标渲染

1. `PaneView::render_active_content` 读取活动 Tab 状态。
2. `PaneView::render_viewer_content` 按软换行段构建可视行。
3. `selected_column_range_for_line` 计算行级选区。
4. `selected_range_in_wrapped_segment` 切分为段级选区。
5. `render_wrapped_line_content` 逐段渲染选中背景与光标。

对应文件：
`crates/zom-gpui/src/components/pane/view.rs`

## 3. 默认快捷键从哪里来

1. `zom-protocol::command::kind` 维护命令语义目录（`id/title/description`）。
2. `zom-input::defaults` 维护默认快捷键声明（`CommandKindId + scope + keystroke`）。
3. `zom-input::defaults::command_from_kind_id` 负责把稳定 ID 映射到可执行 `CommandInvocation`。
4. `build_default_shortcut_registry` 注册默认绑定，`default_keymap` 生成解析映射。

对应文件：
`crates/zom-protocol/src/command/kind/specs/*`  
`crates/zom-protocol/src/command/kind/mod.rs`  
`crates/zom-input/src/defaults.rs`  
`crates/zom-input/src/lib.rs`

说明：这条链路确保“语义目录（protocol）”与“输入策略（zom-input）”边界清晰，且仍由稳定 `CommandKindId` 保持一致性。

## 4. 新增命令的最小改动清单

### 第一步：协议层声明（必做）

1. 在 `EditorAction` 或 `WorkspaceAction` 增加动作枚举。
2. 在 `CommandKind` 增加稳定语义项。
3. 在对应 `specs/*` 增加 `CommandKindSpec`（含 `id/title/description`）。

相关文件：
`crates/zom-protocol/src/command/invocation/*`  
`crates/zom-protocol/src/command/kind/types.rs`  
`crates/zom-protocol/src/command/kind/specs/*`

### 第二步：输入映射接入（按需）

1. 若该命令需要默认快捷键，在 `zom-input/src/defaults.rs` 增加对应键位声明。
2. 若该命令可由快捷键直接触发，补齐 `CommandKindId -> CommandInvocation` 映射分支。

相关文件：
`crates/zom-input/src/defaults.rs`

### 第三步：执行层实现（按领域）

1. Workspace 命令：补到 `handle_workspace_command` 或其子分发函数。
2. Editor 命令：补到 `apply_action`，必要时补事务函数。

相关文件：
`crates/zom-runtime/src/state/desktop_app.rs`  
`crates/zom-editor/src/features/editing/invocation.rs`

### 第四步：渲染或交互接入（按需要）

1. 若只需键盘触发，可不改 GPUI 交互层。
2. 若需要鼠标入口，在 GPUI 事件里调用 `state.handle_command(...)`，并随后 `sync_child_views(...)`。

相关文件：
`crates/zom-gpui/src/root_view/*`  
`crates/zom-gpui/src/components/*`

### 第五步：测试闭环（建议）

1. 协议层：语义目录测试（`command_kind`/`command_meta` 一致性）。
2. 输入层：快捷键映射测试（`resolve_default` 结果）。
3. 执行层：状态变更测试（光标、选区、文本）。
4. 运行层：端到端行为测试（`DesktopAppState`）。

---

## 5. 常见排查路径

1. 按键按下无反应：先看 `to_core_keystroke` 是否能识别该键。
2. 识别了但没命令：看 `zom-input::resolve_default` 与 `Keymap::resolve`。
3. 命令发出了但状态没变：看 `handle_command` 分发分支是否覆盖。
4. 状态变了但 UI 没更新：看是否执行了 `sync_child_views`。
5. 选区有值但没高亮：看 `selected_column_range_for_line` 与 `render_wrapped_line_content`。
