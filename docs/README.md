## 4. 去重与冲突处理规则

1. 同一主题只能有一个 SoT 文档，其他文档只保留摘要与跳转。
2. 若内容冲突，按本页 SoT 表优先级处理。
3. 需要跨主题改规则时，应同步更新 SoT 文档与本页映射。

---

## 5. 维护约定

1. 新增文档时，必须在本页补充“主题 -> SoT”映射。
2. PR 若修改了架构边界或治理规则，需说明是否触及：
   - [架构哲学与模块宪章.md](./architecture/架构哲学与模块宪章.md)
   - [Crate边界契约.md](./architecture/Crate边界契约.md)
3. 架构变化不单独写决策文档，统一通过 [PR 模板](../.github/pull_request_template.md) 留痕。
4. 已废弃文档应在文件顶部标注“Deprecated”并指向替代文档，再择期删除。
